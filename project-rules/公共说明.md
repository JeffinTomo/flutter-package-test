## 项目结构

```
${project-name}
├── public/               # 静态资源
├── src/                  # 前端源码
├── src/router              # 路由定义
├── src/component           # 公共组件
├── src/const              #  静态变量配置
├── src/pages              # 路由的具体实现
├── src/assets              #   图片等静态资源
├── src/hooks              #  全局的 api 及 hooks
├── README.md              #  开发说明
└── cookbook.md             # 使用说明
```


## 命名与注释
1. 不得使用中文
2. 所有函数的输入输出必须有严格的注释或者 interface 说明

## css 部分
1. 方法遵守 BEM https://www.cythilya.tw/2018/06/05/css-methodologies/
2. 尽量使用 taiwind
3. 需要核对实现，避免过多的冗余
4. svg 文件需要单独定义好再引用，不得直接放在代码逻辑中
5. 布局不得使用 absolute

## typescript 部分
1. 不允许出现 any
2. interface 的定义要放入单独的文件，最好是全局的统一文件

## 模板部分
1. 注意抽象 component
2. 模块化拆分，模块在独立的前提下，尽量小
3. 控制到 400 行以内 
4. 重要业务模块，请定义唯一且不变的 id, 命名要突出业务本身

## API 与数据
参考 ./api-hooks
1. api 封装成 service，并通过 hooks 暴露给业务层
2. 常用交互事件也封装 hooks
